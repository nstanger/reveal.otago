Using ICT

The march of ICT progress

  • In particular, storage capacity and network bandwidth are increasing at an amazing rate, with corresponding drops in price.
  • We’ve gone from standalone, isolated applications that weren’t even physically connected to software that is highly integrated and requires a network to run. With wireless, access is available almost anywhere.
  • Changes to business structures have pushed decision making further down the management hierarchy, driven by smaller staffing levels and the availability of cheap, powerful computers and ubiquitous, high capacity networking and storage. This has created a greater demand for information at all levels of the business, and thus a need to collect and store more data. These data are accessed from almost any location.

Generic information
system architecture

  1. Every information system, regardless of how it’s constructed, can be decomposed into the three very high-level components shown. In some systems these components will be very explicit, while in others the boundaries may be so fuzzy as to be almost indistinguishable. In simpler, non-distributed systems, the network component may be negligible or even non-existent.
  2. The front end is what end-users interact with. Typical functions include application processing, data validation, user interface, etc.
  3. The back end manages access to the data resources of the information system. Typical functions include data management, data validation, etc.
  4. The network component initiates, maintains and manages the connection between the front and back ends.
  5. In INFO 214 we will focus on the back end and network components. INFO 221 will cover some aspects of the front end.
